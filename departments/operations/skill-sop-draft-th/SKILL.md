# Thai SOP Draft

Turn messy process notes into a readable Thai SOP.

## Output requirements
- Write in Thai
- Keep steps clear and sequential
- Make it usable by non-technical staff
- Separate required steps, checks, and exceptions when possible
- Do not add fake policy details or approvals that were not provided

## Working method
1. Identify the purpose of the process
2. List steps in logical order
3. Clarify roles or owners if known
4. Add checkpoints or exceptions when relevant
5. Format for day-to-day use

## Default output structure
1. SOP objective
2. scope or when to use
3. step-by-step process
4. checks / cautions
5. escalation or exception handling if available
